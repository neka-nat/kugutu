# Charpack v0

`.charpack` is the single-file runtime artifact for embedding Kugutu characters in apps.

It is JSON in v0 so it can be inspected, diffed, and generated by agents.

## Top-level shape

```json
{
  "packVersion": "0.1.0",
  "source": {},
  "bundle": {},
  "assets": [
    {
      "id": "primary-svg",
      "type": "svg",
      "content": "<svg>...</svg>"
    },
    {
      "id": "source-svg",
      "type": "svg",
      "content": "<svg>...</svg>"
    }
  ],
  "partAssets": {}
}
```

## Runtime usage

```ts
import { Kugutu } from "@kugutu/runtime-web";

const player = await Kugutu.load("/assistant-mascot.charpack", "#stage");

player.lookAt({ x: 0.2, y: -0.1 });
player.setEmotion("happy", 0.8);
player.setMouthOpen(0.4);
```

`Kugutu.load` accepts a URL or a `CharPack` object, and a selector or an
`HTMLElement`. The lower-level
`createCharacterPlayerFromPack(pack, container, options?)` is also available.

### `parts.selections` is the source of truth for the initial look

At player creation the runtime applies `bundle.parts.selections` to the SVG
(variant visibility, per-part transform, color). So a pack whose selections
were edited *after* compose — a generated charpack, a hand-tuned JSON, an
agent picking parts — renders its selected parts from `Kugutu.load()` alone;
no replayed `setPart()` calls are needed. For a normally compiled pack (SVG
already matches the selections) this is an idempotent no-op. A selection
naming a part that has no variant group in the SVG (composed against an older
catalog) logs a console warning and keeps the baked look for that slot.

## Build

```bash
kugutu pack character.json avatar.base.svg --out assistant-mascot.charpack
```

## Notes

- `bundle` is the same runtime control data produced by `build`.
- `primary-svg` contains the composed SVG with part transforms baked in.
- `source-svg` contains the editable base SVG used when Studio re-opens the pack.
- `source` is included by default so Studio and agents can re-open the pack later.
- `partAssets` (optional) maps catalog part ids to their raw SVG fragments when
  the character uses anchor-mounted file parts, so editors can recompose the
  SVG on part changes without re-reading the source tree.
- Use `--no-source` when distributing a smaller runtime-only pack.
